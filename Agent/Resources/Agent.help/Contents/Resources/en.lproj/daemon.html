The Privileged Helper

What It Is

Agent! ships with a small command-line tool called AgentHelper that runs as a Launch Daemon — meaning it runs as root and survives reboots. Its only job is to execute shell commands on behalf of the main app, over an authenticated XPC connection.

Why It's Needed

Most tasks run as your user account. But some operations — disk cloning, system configuration, installing software to /usr/local — require root. Rather than asking for an admin password every time, Agent! asks once, installs the helper, and the helper does the root-only work on request.

Installing the Helper

  1. Open Settings (⌘,)
  2. Go to the Daemon tab
  3. Click Install Helper
  4. macOS will show a notification — click Open System Settings
  5. In Login Items & Extensions, enable the Agent helper

Removing the Helper

From the same Settings tab, click Uninstall Helper. This unloads the daemon and removes its registration.

The helper only accepts connections from the signed Agent! app bundle. It verifies the caller's code signature before executing anything. If Agent! is modified or tampered with, the XPC connection is refused.

What the Helper Will Not Do

The same destructive-command guardrails that protect the user shell also protect the root shell. Commands like rm -rf /, writes to raw disk devices, and fork bombs are refused before they reach the shell, no matter which path invokes them.

Technical Details

  • Mach service: Agent.app.toddbruss.helper
  • Protocol: HelperToolProtocol (execute, cancel)
  • Registration: SMAppService.daemon
  • Plist location: /Library/LaunchDaemons/Agent.app.toddbruss.helper.plist (installed by SMAppService)
