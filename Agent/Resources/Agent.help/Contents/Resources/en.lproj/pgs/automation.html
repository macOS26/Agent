Automating Other Apps

Accessibility (preferred for UI)

The accessibility tool uses AXorcist to find and act on UI elements by role and title — no coordinates, no screenshots. Examples:

  • Click a button: find an AXButton by title and click it
  • Type into a field: find an AXTextField and type into it
  • Invoke a menu: click_menu_item with a path like "File > Save"
  • Read what's on screen: find_element returns the element's property dump
Pass either a natural app name ("Photo Booth") or a bundle ID ("com.apple.PhotoBooth") — both resolve correctly.

AppleScript

Every Mac app has its own AppleScript vocabulary. Agent! always reads the app's SDEF (Scripting Definition) first, then writes verified code. About 50 SDEFs ship bundled (Music, Mail, Safari, Pages, Finder, System Events, etc.).

JavaScript for Automation (JXA)

JXA is the JavaScript surface over the same scripting dictionaries. Use it when JS syntax is clearer than AppleScript's.

Agent Scripts

For complex automation, Agent! can write Swift dylibs — stored at ~/Documents/AgentScript/agents/. These run in-process with full TCC, use ScriptingBridge for strongly-typed app automation, and can mix AppleScript where it's shorter.

System Events

com.apple.systemevents is the universal helper. It handles keystrokes, menu invocation, UI element clicks, and process management for apps whose own dictionary doesn't cover what you need.
