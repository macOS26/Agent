Troubleshooting

Agent! Isn't Clicking or Typing

Accessibility permission is missing or was denied. Open System Settings → Privacy & Security → Accessibility, remove Agent! from the list, then re-run a task that needs it and grant permission when prompted.

"The helper tool is not installed"

Open Settings → Daemon and click Install Helper. If the install button does nothing, check System Settings → Login Items & Extensions — the helper may be waiting for you to enable it.

Build Fails in Xcode

  • Make sure Agent.xcodeproj is selected, not a nested project
  • Development team must be set to 469UCUB275 (or your own team)
  • Deployment target is macOS 26.0, Swift 6.2
  • Clean the build folder (⌘⇧K) and rebuild

API Key Errors

Open Settings → Providers and verify the key is pasted correctly (no leading/trailing spaces). For Claude keys, they begin with sk-ant-.

Ollama Models Not Listed

Make sure Ollama is running (ollama serve) and at least one model is pulled (ollama pull llama3.1). Then click Refresh Models in Settings.

Tasks Get Stuck or Loop

Press ⌘. to stop. Agent! has built-in loop detection, but if the model is truly stuck, starting a fresh tab with a clearer prompt usually works better than pushing through.

Reset Everything

To return Agent! to defaults:

  1. Quit Agent!
  2. Remove ~/Library/Preferences/Agent.app.toddbruss.plist
  3. Remove ~/Library/Application Support/Agent/
  4. Relaunch
This does not remove saved scripts in ~/Documents/AgentScript/. Delete that folder separately if you want a completely fresh start.
